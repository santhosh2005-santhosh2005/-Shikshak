# DysCover Project Manifesto

## 🚀 Mission
Transform DysCover from a generic screening interface into a role-based, human-centered adaptive learning platform — while preserving all existing functionality.

## ⚖️ Core Rules
- **Do NOT modify or break** any existing features or logic.
- **Do NOT change** assessment, learning mode, dashboard, parent digest, or progress tracking logic.
- **Only ENHANCE UI/UX** by restructuring layout and adding new UI layers.
- **Maintain current Tailwind styling system** and reuse components wherever possible.

## 🎭 User Roles
1. **Child**: Focused on "Learning Your Way" with large buttons, friendly tone, and icon-based feedback.
2. **Teacher**: Minimal, card-based dashboard for tracking weak areas and suggested actions.
3. **Parent**: Emotionally simple "Today's Report" with bullet points and suggestions.

## 📖 Feature Enhancements (UI Only)

### 1. Dyslexia Mode
- Large readable text.
- OpenDyslexic font support.
- Visible audio/TTS buttons.
- Word highlighting on hover/playback.

### 2. ADHD Mode
- Big, satisfying buttons.
- Clear timers.
- Reward feedback (e.g., "+10 ⭐").
- Single-task focus (one question per screen).

### 3. Autism Mode
- Progress indicators ("Step X of Y").
- High layout consistency.
- Predictable UI transitions.
- Calm spacing and structure.

## 🎨 Visual Identity
- **Primary Messaging**: "Personalized Learning for Every Child"
- **Subtext**: "We understand how your child learns and adapt education to match."
- **Tone**: Friendly, supportive, human-centered.
- **Design System**: Soft color palette with cinematic transitions (Calm vs. Neo-Brutalism).
